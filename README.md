# AI 实践案例报告 Skill

一个用于整理 AI 实践案例的 Skill / Prompt 工作流。

它可以把用户零散描述的 AI 使用经历，整理成一份结构清晰、信息密度高、适合提交或展示的案例报告。适合用于活动投稿、案例征集、内部知识沉淀、社群分享和个人作品整理。

这个项目原生支持 Codex Skill 使用，同时也可以迁移到 Claude、ChatGPT、自定义 GPT、本地模型或其他 Agent 框架中使用。

## 它解决什么问题

很多人确实做过有价值的 AI 实践，但在提交案例时容易卡住：

- 不知道怎么把经历写成案例；
- 只会说“我用了 AI”，讲不清原来的问题和变化；
- 不知道哪些信息够成案，哪些还需要补充；
- 容易写成工具介绍、宣传稿或流水账；
- 缺少改造前后对比、最终效果和可复用经验。

这个 Skill 的目标是降低用户提交案例的门槛。用户只需要回答几个具体问题，Codex 会判断信息是否足够，不够就继续追问，足够后生成正式案例报告。

## 适用对象

适合以下用户使用：

- 想提交 AI 实践案例，但不知道怎么写的人；
- 活动运营、社群运营、案例征集负责人；
- 希望沉淀 Prompt、Skill、Agent 或自动化工作流的人；
- 用 AI 做过开发、运营、产品、销售、客服、内容、设计、研究或个人效率实践的人；
- 想把一次 AI 使用经验整理成可复用方法的人。

## 支持的案例类型

- AI Skill / Prompt 沉淀
- AI 编程与开发提效
- Agent / 自动化工作流搭建
- AI 改造业务 / 工作流程
- AI 小工具 / 小应用开发

如果案例更像自动化、低代码或流程改造，而不是严格的大模型 AI 实践，Skill 也会如实处理，不会强行包装。

## 达成效果

使用后通常可以得到一份包含以下内容的案例报告：

- 案例标题；
- 一句话简介；
- 案例类型；
- 背景与痛点；
- 解决方案概览；
- 改造前后对比表；
- 具体流程；
- 关键 Prompt / Skill / 工作流 / 方法；
- 最终效果；
- 可复用经验与注意事项；
- 相关材料；
- 作者信息。

报告会优先保留用户真实细节，不编造数据，不夸大效果。没有量化数据时，会用可感知的定性变化来表达，例如“从不可编辑图片变成可编辑 PPTX”，而不是空泛地写“效率大幅提升”。

## 工作方式

Skill 会按下面流程工作：

1. 引导用户填写基础信息模板；
2. 判断案例信息完整度；
3. 信息不足时继续追问，每轮最多 5 个问题，最多 3 轮；
4. 信息足够后生成结构化案例报告；
5. 如果当前环境支持联网搜索，会先做相似案例辅助检查。

## 兼容性说明

这个项目的核心是 `SKILL.md`。它本质上是一套结构化提示词和工作流说明，不只限于 Codex。

| 平台 | 使用方式 |
|---|---|
| Codex | 放入 `~/.codex/skills/` 后，通过 `$ai-case-report` 调用 |
| Claude | 将 `SKILL.md` 内容导入 Claude Skill、项目指令或自定义指令 |
| ChatGPT / GPTs | 将 `SKILL.md` 内容复制到自定义 GPT 的 Instructions |
| 本地模型 | 将 `SKILL.md` 作为 system prompt 或工作流提示词加载 |
| 其他 Agent 框架 | 将 `SKILL.md` 改写成对应框架的角色说明、流程规则或任务模板 |

`agents/openai.yaml` 主要用于 Codex 的界面展示，其他平台可以忽略。

## Codex 安装方法

下载或 clone 这个仓库后，把整个文件夹放到你的 Codex Skills 目录：

```text
~/.codex/skills/ai-case-report
```

目录结构应保持为：

```text
ai-case-report/
├── README.md
├── SKILL.md
└── agents/
    └── openai.yaml
```

然后重启 Codex，或新开一个 Codex 对话。

## Codex 使用方法

在 Codex 新对话里输入：

```text
$ai-case-report
```

然后可以直接说：

```text
我想提交一个 AI 实践案例，帮我引导填写并生成报告。
```

也可以直接粘贴案例信息：

```text
$ai-case-report

【我解决的问题】
我原来……

【使用场景】
……

【使用工具】
……
```

## 输入示例

```text
我用 image2 先生成 PPT 页面图，再用 Codex 根据图片和提示词重建成可编辑 PPTX。
原来 AI 生成的 PPT 只能看不能改，现在可以继续修改标题、正文、数据和图片。
```

Skill 会根据这类描述继续追问，例如使用场景、具体流程、最终效果、可复用经验和作者信息，然后生成正式报告。

## 其他平台使用方法

如果你使用的不是 Codex，可以这样迁移：

1. 打开 `SKILL.md`；
2. 复制其中的完整内容；
3. 粘贴到目标平台的系统提示词、自定义指令、项目指令或 Agent 角色说明里；
4. 向模型输入一段 AI 实践描述，让它按 `SKILL.md` 的流程追问并生成报告。

最简使用方式：

```text
请按照下面这份 AI 实践案例生成规则工作。

<粘贴 SKILL.md 内容>

我想提交一个 AI 实践案例，请先引导我填写信息。
```

## 设计原则

- 真实优先，不编造用户没有提供的信息；
- 不编造时间、效率、成本等数据；
- 信息不足先追问，而不是直接硬写；
- 表达克制，不写成夸张宣传稿；
- 保留真实工具、流程、Prompt 和产物；
- 强制包含改造前后对比；
- 区分事实和建议；
- 默认生成精简、高信息密度版本。

## 注意事项

- 如果案例涉及企业内部数据、客户信息、合同、价格、账号或未公开项目，请在提交前自行确认是否可以公开。
- 如果没有具体数据，可以写定性效果，Skill 不会强迫用户提供数字。
- 如果没有截图、Prompt、配置或项目链接，也可以先写“暂无”。
- 如果想要更长的版本，可以明确要求“生成详细版报告”。

## 文件说明

- `SKILL.md`：Skill 的核心说明文件，Codex 会根据它执行案例整理流程。
- `agents/openai.yaml`：Codex 的界面展示信息。
- `README.md`：给人看的项目说明。
